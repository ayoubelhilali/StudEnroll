🔄 Request/Response Lifecycle (1-10)

1

User UI Request Trigger

The student clicks "Enroll" in the React catalog viewport inside the client web browser (port 5173).

2

Axios HTTP JSON Gateway Call

Axios sends the request payload (EnrollmentRequest DTO) to the API Gateway (port 8060).

3

Eureka Instance Registry Lookup

API Gateway queries Eureka Service Registry (port 8761) to resolve the active instance mapping for enrollment-service.

4

Gateway Forwarding to Service

The API Gateway routes the HTTP POST request to the load-balanced enrollment-service instance (port 8083).

5

WebClient Verification Check

EnrollmentService uses StudentClient & CourseClient (WebClient wrappers) to query student-service (8081) and course-service (8082) concurrently to verify entities exist.

6

SQL Persistence Commit

Upon verification and constraint checks (max 3 students), EnrollmentRepository persists the record in the isolated enrollment_db MySQL database schema.

7

Database Return Records

The databases commit the transaction and return raw confirmation objects back to the service Repository layers.

8

Service Response Packaging

EnrollmentService maps entity records to the expected EnrollmentResponseDTO format (including the 24h deletable status).

9

Final JSON Response Delivery

The microservice sends the JSON payload back through the API Gateway route proxy to the awaiting client connection.

10

UI State Update & Render

React captures the returned JSON payload, resolves the Axios promise, updates client state hook contexts, and dynamically updates the DOM to show the new enrollment.
